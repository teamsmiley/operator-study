# Kubernetes Operator 학습 목표

## 한 줄 요약

Kubernetes에 "나만의 리소스"를 만들고, 그걸 감시하는 프로그램을 만드는 것.

## 비유

학교 급식 시스템으로 비유하면:

- **Kubernetes** = 학교
- **Pod, Deployment** = 급식, 교실 등 학교에 원래 있는 것들
- **CRD** = "동아리"라는 새로운 개념을 학교에 등록하는 것
- **CR** = 실제로 "코딩 동아리"를 신청서로 만드는 것
- **Controller** = 신청서가 들어오면 교실 배정, 선생님 배치를 자동으로 해주는 담당자

## 단계별 학습 목표

### 1단계: CRD와 CR 이해하기

- [ ] CRD가 뭔지 (새로운 리소스 종류를 등록)
- [ ] CR이 뭔지 (등록된 종류의 실제 인스턴스를 생성)
- [ ] CRD만 등록하고, CR을 만들어보기 (Controller 없이)
- [ ] CR을 만들면 뭐가 일어나는지 확인하기

### 2단계: Controller 이해하기

- [ ] Controller가 뭔지 (CR을 감시하고 뭔가를 해주는 프로그램)
- [ ] Reconcile Loop가 뭔지 (원하는 상태와 현재 상태를 맞추는 반복)
- [ ] Controller를 실행하고, CR을 만들면 뭐가 달라지는지 확인하기

### 3단계: 합쳐서 이해하기

- [ ] CRD + CR + Controller = Operator
- [ ] 전체 흐름을 처음부터 끝까지 한번에 해보기
