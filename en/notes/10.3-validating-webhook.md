# 09-3. Validating Webhook -- Rule Validation on CR Create/Update

## What Is a Validating Webhook?

An HTTP endpoint that validates whether a CR conforms to rules and **allows or denies** the request when a CR is created or updated.
It runs after the Mutating Webhook.

Analogy: A security guard. No badge, no entry.

```text
User submits CR create request
  -> Mutating Webhook: auto-fills missing values (already implemented)
  -> Validating Webhook: validates against rules
     -> Pass: stored in etcd
     -> Deny: error returned, not stored
```

## When to Use It

| Use case                   | Example                                                  |
| -------------------------- | -------------------------------------------------------- |
| Field value validation     | Image must include a tag                                 |
| Cross-field comparison     | Cannot reduce replicas by more than 50% at once          |
| Deletion prevention        | CRs with a specific label cannot be deleted              |
| External reference validation | Verify the image is in an allowed registry list        |

## Scaffold Generation

Add Validating Webhook to an existing Mutating Webhook:

```bash
kubebuilder create webhook --group apps --version v1 --kind SimpleApp --programmatic-validation
```

Validating-related code is added to the existing `simpleapp_webhook.go`.

## Three Generated Functions

| Function           | When called    | Role                                                    |
| ------------------ | -------------- | ------------------------------------------------------- |
| `ValidateCreate()` | On CR creation | Validates the create request                            |
| `ValidateUpdate()` | On CR update   | Validates the update request (can compare old and new values) |
| `ValidateDelete()` | On CR deletion | Validates the delete request                            |

ValidateUpdate receives both `oldObj` and `newObj`, allowing comparison between the previous and new values.
This is a key capability that CRD validation markers cannot provide -- only Webhooks can do this.
