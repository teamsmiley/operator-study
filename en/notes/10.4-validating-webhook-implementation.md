### After (logic added)

```go
func (v *SimpleAppCustomValidator) ValidateCreate(_ context.Context, obj *appsv1.SimpleApp) (admission.Warnings, error) {
    simpleapplog.Info("Validation for SimpleApp upon creation", "name", obj.GetName())

    // Image must include a tag
    if !strings.Contains(obj.Spec.Image, ":") {
        return nil, fmt.Errorf("image must include a tag (e.g., nginx:1.25), got: %s", obj.Spec.Image)
    }

    return nil, nil
}
```

ValidateUpdate uses the same validation logic,
but validates `newObj` (the new value) rather than `oldObj` (the previous value).

## How It Works

```text
User input: image: nginx        -> Mutating converts to nginx:latest -> Validating passes
User input: image: nginx:1.25   -> Mutating makes no change -> Validating passes
```

Since Mutating runs first, an image without a tag gets `:latest` appended by the Mutating Webhook.
By the time it reaches Validating, the tag is already present. This acts as a double safety net.

## Return Values

| Return value                  | Meaning                                            |
| ----------------------------- | -------------------------------------------------- |
| `return nil, nil`             | Validation passed (CR storage allowed)             |
| `return nil, fmt.Errorf(...)` | Validation failed (CR storage denied, error returned) |
| `return warnings, nil`        | Validation passed but with warning messages        |

## Complete Flow: Mutating and Validating Together

```text
User submits CR create/update request
  -> Mutating (Default function): auto-fills missing values
  -> Validating (ValidateCreate/Update function): validates against rules
     -> Pass: stored in etcd -> Controller's Reconcile is called
     -> Deny: error returned, not stored
```

## Structural Comparison

|                  | Mutating                         | Validating                                           |
| ---------------- | -------------------------------- | ---------------------------------------------------- |
| Struct           | SimpleAppCustomDefaulter         | SimpleAppCustomValidator                             |
| Core functions   | Default()                        | ValidateCreate(), ValidateUpdate(), ValidateDelete() |
| Return value     | error (mutation success/failure) | (Warnings, error) (warning + allow/deny)             |
| What it can do   | Modify values                    | Allow/deny decisions                                 |
| What it cannot do | Deny (returning an error prevents the mutation itself) | Modify values                          |
