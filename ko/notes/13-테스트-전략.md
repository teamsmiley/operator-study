# 테스트 전략

## 한 줄 요약

kubebuilder는 테스트 파일을 자동으로 scaffolding해 준다.
개발자는 TODO 부분에 자기 로직에 맞는 테스트를 채워넣으면 된다.

---

## kubebuilder가 생성하는 테스트 파일

```text
internal/controller/
    suite_test.go                   -- envtest 환경 설정 (수정 불필요)
    simpleapp_controller_test.go    -- Reconcile 테스트 (여기를 수정)

internal/webhook/v1/
    webhook_suite_test.go           -- Webhook 환경 설정 (수정 불필요)
    simpleapp_webhook_test.go       -- Webhook 테스트 (여기를 수정)

test/e2e/
    e2e_suite_test.go               -- E2E 환경 설정 (수정 불필요)
    e2e_test.go                     -- E2E 테스트 (여기를 수정)
```

`suite_test.go` 류의 파일은 테스트 환경(envtest, Ginkgo)을 설정하는 파일이다.
kubebuilder가 완성된 상태로 제공하므로 건드릴 일이 거의 없다.

개발자가 수정해야 하는 파일은 3개다:

| 파일                           | 무엇을 테스트하나 | 클러스터 필요?     |
| ------------------------------ | ----------------- | ------------------ |
| `simpleapp_controller_test.go` | Reconcile 로직    | 아니오 (envtest)   |
| `simpleapp_webhook_test.go`    | Webhook 로직      | 아니오             |
| `e2e_test.go`                  | 전체 배포 + 동작  | 예 (진짜 클러스터) |

---

## 테스트 프레임워크

kubebuilder는 **Ginkgo + Gomega** 조합을 사용한다.

```go
// Ginkgo + Gomega 스타일
var _ = Describe("SimpleApp Controller", func() {
    It("should successfully reconcile", func() {
        Expect(err).NotTo(HaveOccurred())
    })
})

// Go 표준 스타일 (비교용)
func TestReconcile(t *testing.T) {
    if err != nil {
        t.Fatalf("unexpected error: %v", err)
    }
}
```

| 프레임워크 | 역할                     | Go 표준과 비교   |
| ---------- | ------------------------ | ---------------- |
| Ginkgo     | BDD 스타일 테스트 실행기 | `testing.T` 대체 |
| Gomega     | 가독성 좋은 Assertion    | `assert` 대체    |

## 1. simpleapp_controller_test.go -- Reconcile 테스트

### envtest란?

**가짜 쿠버네티스 API 서버**를 로컬에서 띄워서 테스트하는 도구다.
진짜 클러스터 없이도 CRD 등록, CR 생성, Reconcile 호출이 가능하다.

```text
envtest 환경:
    etcd (내장)  +  kube-apiserver (내장)
        ↑
    테스트 코드에서 k8sClient로 접근
```

실제 클러스터와의 차이:

| 있는 것   | 없는 것        |
| --------- | -------------- |
| API 서버  | kubelet        |
| etcd      | 실제 Pod 실행  |
| CRD 등록  | 네트워크, 볼륨 |
| CRUD 동작 | 스케줄링       |

### 코드

```go
var _ = Describe("SimpleApp Controller", func() {
    Context("When reconciling a resource", func() {

        BeforeEach(func() {
            // Spec을 채워서 CR 생성 (image, replicas 필수)
            replicas := int32(1)
            resource := &appsv1.SimpleApp{
                ObjectMeta: metav1.ObjectMeta{
                    Name:      "test-resource",
                    Namespace: "default",
                },
                Spec: appsv1.SimpleAppSpec{
                    Image:    "nginx:latest",
                    Replicas: &replicas,
                },
            }
            Expect(k8sClient.Create(ctx, resource)).To(Succeed())
        })

        AfterEach(func() {
            // Deployment 정리 (Reconcile이 생성했을 수 있음)
            deploy := &k8sappsv1.Deployment{}
            err := k8sClient.Get(ctx, typeNamespacedName, deploy)
            if err == nil {
                Expect(k8sClient.Delete(ctx, deploy)).To(Succeed())
            }

            // SimpleApp 삭제
            resource := &appsv1.SimpleApp{}
            err = k8sClient.Get(ctx, typeNamespacedName, resource)
            if err != nil {
                return
            }
            Expect(k8sClient.Delete(ctx, resource)).To(Succeed())

            // Reconcile 호출하여 Finalizer 정리 → CR이 실제로 삭제됨
            controllerReconciler := &SimpleAppReconciler{
                Client: k8sClient, Scheme: k8sClient.Scheme(),
            }
            controllerReconciler.Reconcile(ctx, reconcile.Request{
                NamespacedName: typeNamespacedName,
            })
        })

        // 테스트 1: Reconcile이 에러 없이 완료되는지
        It("should successfully reconcile the resource", func() {
            controllerReconciler := &SimpleAppReconciler{
                Client: k8sClient,
                Scheme: k8sClient.Scheme(),
            }
            _, err := controllerReconciler.Reconcile(ctx, reconcile.Request{
                NamespacedName: typeNamespacedName,
            })
            Expect(err).NotTo(HaveOccurred())
        })

        // 테스트 2: Reconcile 후 Deployment가 생성되었는지
        It("should create a Deployment after reconcile", func() {
            controllerReconciler := &SimpleAppReconciler{
                Client: k8sClient,
                Scheme: k8sClient.Scheme(),
            }
            _, err := controllerReconciler.Reconcile(ctx, reconcile.Request{
                NamespacedName: typeNamespacedName,
            })
            Expect(err).NotTo(HaveOccurred())

            // Deployment가 생성되었는지 확인
            deploy := &k8sappsv1.Deployment{}
            err = k8sClient.Get(ctx, typeNamespacedName, deploy)
            Expect(err).NotTo(HaveOccurred())

            // Deployment 이름과 Container 수 검증
            Expect(deploy.Name).To(Equal("test-resource"))
            Expect(deploy.Spec.Template.Spec.Containers).To(HaveLen(1))
        })
    })
})
```

관련 커밋: `0ee757b`

### 핵심 포인트

1. **BeforeEach에서 Spec을 채워야 한다** -- image가 비어있으면 Deployment 생성이 의미 없음
2. **AfterEach에서 Finalizer를 처리해야 한다** -- Reconcile이 Finalizer를 추가하기 때문에,
   CR을 삭제한 후 Reconcile을 한 번 더 호출해서 Finalizer를 제거해야 CR이 실제로 삭제됨
3. **Reconcile 함수를 직접 호출**해서 결과를 검증한다 -- 실제 Manager를 띄우지 않는다

## 2. simpleapp_webhook_test.go -- Webhook 테스트

Webhook의 Defaulter/Validator 함수를 직접 호출해서 검증한다.
API 서버 없이, envtest 없이 실행 가능하다.

### 코드

```go
Context("When creating SimpleApp under Defaulting Webhook", func() {
    It("태그 없는 image에 :latest를 추가해야 한다", func() {
        obj.Spec.Image = "nginx"
        err := defaulter.Default(ctx, obj)
        Expect(err).NotTo(HaveOccurred())
        Expect(obj.Spec.Image).To(Equal("nginx:latest"))
    })

    It("태그가 있는 image는 그대로 유지해야 한다", func() {
        obj.Spec.Image = "nginx:1.25"
        err := defaulter.Default(ctx, obj)
        Expect(err).NotTo(HaveOccurred())
        Expect(obj.Spec.Image).To(Equal("nginx:1.25"))
    })
})

Context("When creating SimpleApp under Validating Webhook", func() {
    It("태그가 있는 image는 생성을 허용해야 한다", func() {
        obj.Spec.Image = "nginx:1.25"
        _, err := validator.ValidateCreate(ctx, obj)
        Expect(err).NotTo(HaveOccurred())
    })

    It("태그가 없는 image는 생성을 거부해야 한다", func() {
        obj.Spec.Image = "nginx"
        _, err := validator.ValidateCreate(ctx, obj)
        Expect(err).To(HaveOccurred())
    })
})

Context("When updating SimpleApp under Validating Webhook", func() {
    It("태그가 있는 image로 수정은 허용해야 한다", func() {
        oldObj.Spec.Image = "nginx:1.24"
        obj.Spec.Image = "nginx:1.25"
        _, err := validator.ValidateUpdate(ctx, oldObj, obj)
        Expect(err).NotTo(HaveOccurred())
    })

    It("태그가 없는 image로 수정은 거부해야 한다", func() {
        oldObj.Spec.Image = "nginx:1.24"
        obj.Spec.Image = "nginx"
        _, err := validator.ValidateUpdate(ctx, oldObj, obj)
        Expect(err).To(HaveOccurred())
    })
})
```

관련 커밋: `b4fc5cf`

### 핵심 포인트

1. **Defaulter 테스트** -- `Default()` 호출 후 obj의 값이 변경되었는지 확인
2. **Validator 테스트** -- `ValidateCreate()`/`ValidateUpdate()` 호출 후 err 유무로 판단
3. **API 서버 불필요** -- 순수 함수 호출이므로 envtest 없이 바로 실행 가능

## 3. e2e_test.go -- E2E 테스트

**진짜 클러스터**에 CRD 설치, Operator 배포, 전체 동작을 검증한다.
kubebuilder가 충실하게 작성해 놓은 상태다.

### 테스트 흐름

```text
BeforeSuite:
    1. make docker-build    (이미지 빌드)
    2. k3d image import     (k3d 클러스터에 이미지 로드)

BeforeAll:
    3. kubectl create ns    (Namespace 생성)
    4. make install         (CRD 설치)
    5. make deploy          (Operator 배포)

테스트:
    6. Operator Pod가 Running인지 확인
    7. 메트릭 엔드포인트가 동작하는지 확인
    8. cert-manager가 인증서를 생성했는지 확인
    9. Mutating Webhook CA가 주입되었는지 확인
   10. Validating Webhook CA가 주입되었는지 확인

AfterAll:
   11. make undeploy        (Operator 제거)
   12. make uninstall       (CRD 제거)
   13. kubectl delete ns    (Namespace 제거)
```

### k3d 사용을 위해 수정한 부분

kubebuilder 기본 E2E는 **Kind**를 전제로 되어 있다.
k3d를 사용하려면 2개 파일을 수정해야 한다:

**e2e_suite_test.go** -- 이미지 로드 방식 변경:

```go
// 변경 전 (Kind)
err = utils.LoadImageToKindClusterWithName(managerImage)

// 변경 후 (k3d)
cmd = exec.Command("k3d", "image", "import", managerImage, "-c", "operator-lab")
_, err = utils.Run(cmd)
```

**e2e_test.go** -- Namespace 중복 생성 에러 방지:

```go
// 변경 전
_, err := utils.Run(cmd)
Expect(err).NotTo(HaveOccurred(), "Failed to create namespace")

// 변경 후
_, _ = utils.Run(cmd) // 이미 존재하면 무시
```

관련 커밋: `6b41af2`

### 사전 조건

- k3d 클러스터가 실행 중이어야 한다
- cert-manager는 테스트가 자동으로 설치한다 (이미 있으면 skip)

### 특징

- `//go:build e2e` 태그가 있어서 일반 `go test`로는 실행되지 않는다
- `kubectl` 명령어를 직접 실행한다
- `make test-e2e`는 Kind를 사용하므로, k3d 환경에서는 직접 실행해야 한다

---

## 실행 명령어 정리

```bash
# Controller + Webhook 테스트 (클러스터 불필요)
make test

# E2E 테스트 -- k3d 환경 (cert-manager 사전 설치 필요)
go test -tags=e2e ./test/e2e/ -v -ginkgo.v

# 특정 테스트만 실행
go test ./internal/controller/... -v
go test ./internal/webhook/... -v

# Ginkgo로 직접 실행 (더 상세한 출력)
ginkgo -v ./internal/controller/...
```

```text
  3. cert-manager 인증서 생성
  4. Mutating Webhook CA 주입
  5. Validating Webhook CA 주입
```

## 정리

```text
make test (로컬에서 빠르게)
├── simpleapp_controller_test.go  -- Reconcile 검증 (envtest)
└── simpleapp_webhook_test.go     -- Defaulter/Validator 검증

CERT_MANAGER_INSTALL_SKIP=true go test -tags=e2e ... (클러스터에서 확실하게)
└── e2e_test.go                   -- 진짜 배포 후 전체 동작 검증
```

핵심: kubebuilder가 테스트 파일을 scaffolding해 준다.
suite 파일은 그대로 쓰고, 테스트 파일 3개에 자기 로직을 채워넣으면 된다.
